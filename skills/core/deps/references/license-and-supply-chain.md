# License and Supply Chain

Check license compatibility, project health, maintenance cadence, vulnerability history, transitive dependencies, bundle or image size, and existing alternatives.

## Additional checks

- Typo-squatting and package provenance.
- Native code, postinstall scripts, binary downloads, or install-time network behavior.
- Maintainer cadence and bus factor.
- License policy: allow, warn, block, unknown.
- Version pinning and lockfile behavior.
- Runtime, build, image, or browser bundle impact.

Escalate unknown license or supply-chain risk before release.
